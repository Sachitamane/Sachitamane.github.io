Real Life Elevator Example

  • The issue with this design is that the users will get confused if the numbers are buttons or if the side panel are the buttons
  • But in reality the white light is the button

 

Why is it good or bad –

  • Bad: the buttons are confusing
  • Good: it has braille letters to aid anyone who is blind
    • the spacing between the buttons is good and easy to understand

Most Common Features of an Elevator

  • most common uses in elevator:

    • buttons for different floors
    • buttons for closing and opening doors

 

  • most rare uses

    • call button
    • bell button
    • panel which accesses the controls

 

  • this interface makes it easier to find the buttons regarding floors and door open and close button

 

  • the common sequence that a user will perform is:
    • find the button regarding floor they want to go
    • to and press the button
    • but for this specific interface it can be a bit confusing since it isn’t clear exactly which one is the button
    • the user will have to use trial and error to find the button
    • first by pressing the number
    • and then maybe the flat panel on the left
  • the interface provides feedback using light and sound, the button lights up red and we hear a sound
  • one of the biggest current mistakes is that the buttons are not intuitive
